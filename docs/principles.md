# 신뢰 약속 7가지 (Trust Principles)

> BOOTCAMP_HY 부동산 신뢰 플랫폼이 사용자/중개사/사회에 지키는 **공개 약속**.
> 이 문서는 사이트 푸터에서도 동일한 내용으로 노출된다.
> 모든 기능 설계와 운영 정책은 이 7가지 원칙에서 어긋나지 않아야 한다.

---

## 0. 왜 이 약속이 필요한가

부동산 거래는 **정보 비대칭** 이 가장 큰 시장 중 하나다.
- 중개사는 시세·매물·법률에 능통하지만, 사용자는 평생 몇 번 안 한다.
- 잘못된 거래의 손실이 크다 (전세사기 평균 피해액 1.5억).
- 기존 플랫폼은 광고 수익 모델이라 **부정 정보 노출을 꺼린다**.

→ 우리 플랫폼은 **사용자 편의보다 사용자 안전을 우선** 한다.
이 차별성이 곧 차별화 포인트.

---

## 원칙 1. 투명한 점수 공식

### 약속
신뢰 점수 산정 공식을 **모두 공개** 한다. 블랙박스 알고리즘 사용 금지.

### 왜
- 점수를 믿게 만드는 유일한 방법은 **계산 과정을 보여주는 것**
- 중개사 입장에서도 "어떻게 점수를 올릴 수 있는지" 알 수 있어야 공정

### 어떻게
- [docs/trust_score_formula.md](trust_score_formula.md) 공식 사이트에 게시
- 중개사 상세 페이지에서 **점수 항목별 breakdown** 표시
  - 예: "법적 신뢰성 30/30 ✓, 운영 안정성 15/15 ✓, 평판 24/30"
- 점수 변경 시 **변경 내역 로그** 사용자에게 표시
- 알고리즘 변경 시 **변경 공지 30일 전 사전 공지**

---

## 원칙 2. 약자 보호 모드 (Vulnerable User Mode)

### 약속
처음 집을 구하는 사람, 노인, 외국인 등 **부동산 지식이 부족한 사용자** 를 특별히 보호한다.

### 왜
- 사기 피해의 80% 이상이 거래 경험 적은 20~30대와 노인층
- 전문 용어를 모르면 위험 신호를 인지 못 함

### 어떻게
- **"처음 집 구해요" 체크박스** → 활성화 시:
  - 모든 전문 용어에 자동 툴팁 (등기부, 근저당, 확정일자 등)
  - 단계별 가이드 강제 노출 (계약 전 → 계약 → 계약 후)
  - 위험 신호 알림 강도 ↑ (배너 + 모달 + 메일)
- 외국어 지원 (영어/중국어/베트남어 — 최소 1개부터 시작)
- 글자 크기 키우기 토글 (고령자 접근성)

---

## 원칙 3. 사기 조기경보 시스템

### 약속
거래 진행 중에 **사기 위험을 자동 감지** 하고 즉시 알림.

### 왜
- 사기는 사후 신고보다 **사전 차단** 이 핵심
- 분산된 공공 데이터를 한 곳에 합치면 위험이 보임

### 어떻게
**조합 신호 알고리즘** (단일 신호로는 경고 X, 2개 이상 조합 시 경고):

| 신호 조합 | 경보 |
|---|---|
| 시세보다 30%↓ + 임대인 다중 소유 5건↑ | 🚨 깡통전세 의심 |
| 등기부 변동 잦음 + 보증보험 거부 | 🚨 이중계약 의심 |
| HUG 사고이력 지역 + 같은 임대인 | 🚨 동일 패턴 사기 의심 |
| 중개사 무자격 + 매물 비공식 채널 | 🚨 무자격 중개 의심 |

→ 경보 발생 시: 사용자에게 **무료 법률 상담 링크 즉시 제공** (대한법률구조공단).

---

## 원칙 4. 소수 의견 보호 (Bayesian Fairness)

### 약속
**리뷰 1~2건으로 점수가 폭락하거나 폭등하지 않는다.**
악의적 평점 조작과 신규 차별을 동시에 막는다.

### 왜
- 별점 1점 1건만 받아도 중개사 평점이 폭락하는 건 부당
- 반대로 별점 5점 1건만으로 신뢰 부여하는 것도 위험

### 어떻게
- **베이지안 평균 공식** 적용 (`trust_score_formula.md` 참고)
- 최소 유의미 리뷰 수: 5건 (그 미만은 "평가 중" 표시)
- 동일 IP / 동일 사용자의 중복 리뷰 자동 차단
- 비정상 패턴 감지 (단기간 다수 리뷰, 동일 문구 등) → 검수 큐

---

## 원칙 5. 신규 중개사의 공정한 기회

### 약속
신규 사업자도 **차별받지 않고 평가받을 기회** 를 받는다.

### 왜
- 운영 기간 가산점은 합리적이지만, 신규를 **0점 처리하면 진입 장벽**
- 좋은 신규 중개사가 기존의 나쁜 관행에 도전할 기회 차단됨

### 어떻게
- 신규(1년 미만) 중개사 → **감점 X, "신규" 뱃지 부여**
- 리뷰 부족(5건 미만) → **점수 옆 "평가 중" 표시**, 사용자에게 안내
- 광고/노출 알고리즘에서 **점수 외 다양성 가중치** 적용
  - 점수 80점 이상 중개사 중 일정 비율은 신규에게 노출 슬롯 배정
- 점수 미달 신규에게 **점수 올리는 방법 가이드** 제공

---

## 원칙 6. 광고와 신뢰 점수 완전 분리

### 약속
**신뢰 점수 순위는 절대 돈으로 살 수 없다.**
광고는 광고로 명확히 표시한다.

### 왜
- 광고 기반 노출 = 사용자 신뢰의 가장 큰 적
- 호갱노노/직방의 한계가 여기에 있음

### 어떻게
- 점수 정렬은 순수 알고리즘 기반, **광고 제외**
- 광고 노출 시:
  - 항상 **"광고" 라벨** 명시 (표시광고법 준수)
  - 광고는 별도 영역에 분리 (검색 결과 상단/하단 별도 박스)
  - 광고 게재 중개사도 점수는 똑같이 계산 (광고비가 점수에 영향 X)
- 수익 모델: 광고 + 프리미엄 기능(중개사용 대시보드 등). **점수 관련 기능은 100% 무료**

---

## 원칙 7. 데이터 공익 환원

### 약속
플랫폼이 누적한 데이터 중 **공익 가치가 큰 부분은 사회에 공개** 한다.

### 왜
- 공공데이터를 활용해 만든 서비스는 가공물도 공익적 가치를 가져야 함
- 부동산 사기 패턴 분석은 학계/정부에 큰 도움이 됨

### 어떻게
- **연 1회 투명성 리포트** 공개 (PDF)
  - 신고 통계, 위험 지역 분석, 사기 패턴 변화 등 (개인정보 제거)
- 학술 연구자 대상 **익명화 데이터셋 무상 제공** (신청 절차 필요)
- 정부/공익기관과의 데이터 공유 협력 (HUG, 국토부 등)
- 다만 **개별 중개사 평가 정보는 공개 X** (영업 기밀 + 명예훼손 방지)

## 부록 A. 신고 및 이의제기 절차

원칙 1, 3, 4 와 직접 연관.

### 사용자 → 중개사 신고
1. 신고 사유 선택 + **증거 자료 첨부 필수** (스크린샷, 계약서 일부 등)
2. 운영자 검수 (영업일 기준 3일 이내)
3. 중개사에게 **소명 기회 7일** 부여
4. 양측 자료 검토 후 판정 → 점수 반영

### 중개사 → 사용자 신고 (악의적 리뷰)
1. 동일 절차 (증거 첨부 필수)
2. 명백한 허위/조작 입증 시 리뷰 삭제 + 점수 복구

### 처리 결과 공시
- 모든 신고 처리 결과는 **익명 통계로 투명성 리포트에 반영**

3. 사기 위험을 사전에 알려드립니다

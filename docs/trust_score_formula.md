# 신뢰 점수 공식 (Trust Score Formula)

> BOOTCAMP_HY 부동산 신뢰 플랫폼의 **중개사 신뢰 점수** 산정 공식 정의 문서.
> 점수는 100점 만점. 70/80/90점 기준으로 리본 등급 부여.
> 모든 가점/감점은 **공개**한다. (`docs/principles.md` 의 투명성 원칙)

---

## 0. 설계 원칙

1. **공공데이터 우선** — 자동 검증 가능한 항목에 큰 가중치
2. **위험 감점 빠르게, 가점은 천천히** — 신뢰는 잃기 쉬워야 자연스러움
3. **블랙박스 금지** — 모든 항목의 산정 근거를 사용자에게 표시
4. **신규 차별 방지** — 베이지안 평균(Bayesian average)으로 리뷰 적은 중개사 보호
5. **AI는 보조** — 공공데이터 기반 점수가 70%, AI는 30% 비중

---

## 1. 점수 구성 (100점 만점)

| 항목 | 비중 | 데이터 출처 | 자동화 |
|---|---|---|---|
| A. 법적 신뢰성 | 30점 | 공공데이터 (브로커 모델 필드) | ★ 완전 자동 |
| B. 운영 안정성 | 15점 | 공공데이터 (등록일자) | ★ 완전 자동 |
| C. 거래 투명성 | 15점 | 외부 공공 API (국세청/국토부) | ★ 자동 |
| D. 고객 응답성 | 10점 | 앱 내부 행동 데이터 | 자동 (앱 사용 후) |
| E. 평판 (리뷰+AI) | 30점 | 사용자 후기 + AI 감정분석 | 반자동 |
| **합계** | **100점** | | |

## 2. 항목별 세부 점수

### A. 법적 신뢰성 (최대 30점)

기존 `brokers.RealEstateAgent` / `brokers.EBBrokerInfo` 모델 필드 그대로 활용.

| 조건 | 점수 | 판정 기준 (필드) |
|---|---|---|
| 영업 상태 정상 | +10 | `RealEstateAgent.sttus_se_code` 가 정상 영업 코드 |
| 공인중개사 자격증 보유 | +10 | `EBBrokerInfo.crqfc_no` 값 존재 + 취득일 유효 |
| 보증보험 유효 기간 내 | +10 | `today` 가 `estbs_begin_de` ~ `estbs_end_de` 사이 |

→ **이 30점은 첫 출시 때부터 자동 계산 가능**. 추가 개발 불필요.

---

### B. 운영 안정성 (최대 15점)

`RealEstateAgent.regist_de` (등록일자) 기준으로 단계별 부여.

| 운영 기간 | 점수 |
|---|---|
| 10년 이상 | +15 |
| 5년 이상 ~ 10년 미만 | +10 |
| 1년 이상 ~ 5년 미만 | +5 |
| 1년 미만 | 0 (감점 X, **"신규" 뱃지 부여**) |

→ 신규 중개사는 차별하지 않음. 대신 "신규" 표시로 사용자에게 정보 전달.

---

### C. 거래 투명성 (최대 15점)

외부 공공 API 연동 필요. **Phase 1~3 에서 단계적 추가**.

| 조건 | 점수 | 출처 |
|---|---|---|
| 국세청 사업자등록 상태 정상 | +5 | api.odcloud.kr |
| 실거래가 신고 누락 의심 없음 | +5 | rt.molit.go.kr |
| 광고 매물과 실거래 일치율 80%↑ | +5 | 내부 매물 데이터 + 실거래가 비교 |

→ 첫 출시는 첫 번째 항목(+5)만 구현. 나머지는 데이터 쌓이면 추가.

---

### D. 고객 응답성 (최대 10점)

앱 내부에서 중개사가 사용자 문의에 응답한 데이터 기반.

| 조건 | 점수 |
|---|---|
| 평균 응답시간 < 1시간 (영업시간 기준) | +5 |
| 응답률 > 90% (최근 30건 기준) | +3 |
| 계약 후 30일 이내 사후 문의 응답 | +2 |

→ **앱에 일정 시간 사용자가 있어야 측정 가능**. 출시 1~2개월 후부터 실제 점수 반영.
출시 직후에는 **N/A (측정 중)** 으로 표시.

---

### E. 평판 - 리뷰 + AI 감정분석 (최대 30점)

#### E-1. 실명 리뷰 평균 (최대 15점)

`interactions.Review.score` (별점 1~5) 활용.

**베이지안 평균 공식** (신규 중개사 차별 방지):

```
보정점수 = (v / (v + m)) × R + (m / (v + m)) × C

  R = 해당 중개사 평균 별점 (1~5)
  v = 해당 중개사 리뷰 수
  m = 최소 유의미 리뷰 수 (기본값 5)
  C = 전체 중개사 평균 별점 (예: 3.5)
```

이렇게 계산한 보정 별점(1~5)을 15점 만점으로 환산:

```
실명 리뷰 점수 = (보정점수 - 1) / 4 × 15
```

예시:
- 별점 5.0, 리뷰 100건 → 거의 5점 그대로 → 15점
- 별점 5.0, 리뷰 1건 → C(3.5)에 가깝게 보정 → 약 9점
- 별점 1.0, 리뷰 1건 → C 쪽으로 보정 → 약 9점 (악의적 1건 리뷰 방지)

#### E-2. 익명 게시판 감정점수 (최대 10점)

**키워드 사전 + AI 모델 하이브리드**.

```
원점수 = (긍정 키워드 합) - (부정 키워드 합) + (AI 긍정확률 × 5)

10점 만점 환산 = max(0, min(10, (원점수 + 10) / 4))
```

자세한 키워드 사전과 AI 모델은 `docs/sentiment_labeling_guide.md` 참고 (작성 예정).

#### E-3. 추천/공감 (최대 5점)

`interactions.Like` 활용. 중개사 프로필 또는 리뷰에 받은 누적 추천.

| 추천 누적 | 점수 |
|---|---|
| 100건 이상 | +5 |
| 50건 이상 | +3 |
| 10건 이상 | +1 |
| 미만 | 0 |

## 3. 위험 감점 (Risk Penalty)

위험 신호 발견 시 즉시 감점. 일부는 **강제 등급 박탈**.

| 위험 신호 | 감점 | 결과 | 출처 |
|---|---|---|---|
| 사기 신고 (검증 완료) | -50 | 모든 리본 박탈, "경고" 표시 | 내부 검수 |
| HUG 전세사고 이력 있음 | -30 | 강제 70점 미만, "경고" 표시 | HUG 공시 |
| 협회 징계 이력 (1년 내) | -15 | 리본 1등급 강등 | KAR 공시 |
| 허위매물 신고 누적 3건+ | -20 | 리본 1등급 강등 | 내부 신고 |
| 폐업 후 재개업 반복 (3회+) | -10 | - | 국세청 데이터 |

### 검수 절차
- 사기/허위매물 신고는 **증거 첨부 필수** + **운영자 검수 후 반영**
- 양방향 이의제기 가능 (`docs/principles.md` 의 신고 정책 참고)

## 4. 리본 등급 (Trust Ribbon)

최종 점수 기준으로 시각적 등급 부여.

| 등급 | 점수 조건 | 색상 | 의미 |
|---|---|---|---|
| 블루 리본 | 90점 이상 + 위험 신호 0건 | 🔵 | 최고 신뢰 |
| 녹색 리본 | 80점 이상 + 위험 신호 0건 | 🟢 | 우수 |
| 노란 리본 | 70점 이상 | 🟡 | 양호 |
| 무등급 | 70점 미만 | - | 신규 또는 평가 부족 |
| 경고 | 사기/HUG 사고 이력 | ⚠️ | 거래 시 주의 |

### 신규 중개사 표시
- 등록 1년 미만 → **"신규" 뱃지** (감점 X, 대신 안내)
- 리뷰 5건 미만 → 점수 옆 **"평가 중"** 표시

---

## 5. 단계별 출시 계획

처음부터 100점 시스템 다 만들면 안 됨. 단계적으로.

### Phase 2 (MVP, 1~2주)
**계산 가능 점수: 45점 (A+B)**
- A. 법적 신뢰성 30점 (브로커 모델 필드 그대로)
- B. 운영 안정성 15점

→ 이 단계만으로도 "노란 리본" 부여 가능 여부 표시 가능.

### Phase 3 (확장, 2~3주)
**+15점 = 60점 가능**
- C. 거래 투명성 +15점 (외부 API 연동)

### Phase 4 (사용자 데이터, 3~4주)
**+40점 = 100점 완성**
- D. 고객 응답성 +10점
- E. 평판 +30점 (리뷰 + AI 감정분석)

### Phase 5 (위험 감점 본격화)
- HUG 사고이력 매핑
- 협회 징계 데이터 수집
- 사기 신고 검수 시스템

## 6. 데이터 모델 (점수 캐싱)

매번 계산하면 느림. 결과를 DB에 저장.

```python
# brokers/models.py 에 추가 예정
class TrustScore(models.Model):
    agent = models.OneToOneField(
        RealEstateAgent,
        on_delete=models.CASCADE,
        related_name="trust_score",
    )
    # 항목별 점수 (디버깅/투명성용)
    score_legal       = models.PositiveSmallIntegerField(default=0)  # A
    score_stability   = models.PositiveSmallIntegerField(default=0)  # B
    score_transparency = models.PositiveSmallIntegerField(default=0) # C
    score_response    = models.PositiveSmallIntegerField(default=0)  # D
    score_reputation  = models.PositiveSmallIntegerField(default=0)  # E
    penalty           = models.SmallIntegerField(default=0)          # 위험 감점

    total      = models.SmallIntegerField(default=0)                  # 합계
    ribbon     = models.CharField(max_length=10, blank=True)          # "blue"/"green"/"yellow"/""
    warning    = models.BooleanField(default=False)                   # 경고 표시 여부

    calculated_at = models.DateTimeField(auto_now=True)
```

→ **일 1회 배치** 로 전체 갱신. 또는 데이터 변동 시 즉시 갱신.

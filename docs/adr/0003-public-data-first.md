# ADR-0003: 공공데이터 우선 정책

## Status

Accepted (2026-05-20)

## Context

BOOTCAMP_HY는 중개사 **신뢰 점수** 를 산정하여 사용자에게 제공하는 플랫폼이다.
점수의 근거가 되는 데이터는 다양한 출처에서 가져올 수 있다.

가능한 데이터 출처는 크게 4가지로 나뉜다:

1. **공공/공식 데이터** — 국토부 실거래가, 국세청 사업자등록, 세움터 건축물대장, HUG 사고이력 등
2. **민간 사이트 크롤링 데이터** — 호갱노노, 직방, 부동산스터디 카페 등
3. **사용자 후기 데이터** — 별점 리뷰, 익명 게시판 글
4. **AI 분석 데이터** — 감정분석 모델로 추론한 점수

각 출처는 신뢰도/법적 안전성/자동화 가능성에서 큰 차이가 있다.

| 출처 | 신뢰도 | 법적 안전성 | 자동화 |
|---|---|---|---|
| 공공/공식 | ★★★ | ★★★ | ★★★ (API) |
| 민간 크롤링 | ★★ | ★ (약관/저작권 이슈) | ★★ |
| 사용자 후기 | ★ (조작 가능) | ★★ | ★★★ |
| AI 추론 | ★ (오류 가능) | ★★ | ★★★ |

**문제 상황**: 어느 데이터를 얼마나 신뢰할지 정하지 않으면,
- 점수 산정 가중치가 매번 흔들린다
- 법적 리스크가 큰 데이터에 의존하다 서비스를 못 하게 될 수 있다
- AI/사용자 데이터에만 의존하면 조작에 취약하다

따라서 **데이터 활용 우선순위에 대한 명확한 정책**이 필요하다.

## Decision

다음 4가지 원칙으로 데이터 활용 우선순위를 확정한다.

### 1. 공공데이터를 1순위 근거로 삼는다

신뢰 점수의 **최소 50% 이상은 공공/공식 데이터 기반**으로 산정한다.
- 국세청 사업자등록 상태
- 국토부 실거래가
- 세움터 건축물대장 / 위반 건축물
- HUG 전세보증사고 이력
- 한국공인중개사협회 자격 정보 (공식 제휴 우선, 어려우면 공시 페이지 활용)

### 2. 민간 크롤링은 보조 자료로만 사용한다

호갱노노, 직방, 부동산스터디 카페 등은 **점수 산정의 주된 근거로 사용하지 않는다.**
- 사용 시 반드시 `robots.txt` 와 사이트 이용약관 확인
- 학습용 데이터(AI 감정분석 학습셋)로는 활용 가능 (조건부)
- 가능하면 **공식 제휴/API** 형태로 전환

### 3. 사용자 후기는 가중치를 제한하고 보정한다

리뷰/게시판 데이터는 **조작 가능성이 크다** 는 점을 인정하고:
- 전체 점수의 30% 이내로 비중 제한 ([trust_score_formula.md](../trust_score_formula.md) 참고)
- 베이지안 평균으로 소수 리뷰 영향 완화
- 동일 IP/사용자 중복 차단

### 4. AI 추론은 마지막에 도입하고, 보조 신호로만 사용한다

AI 감정분석 모델은 **Phase 4 이후 도입** 하며:
- 점수 가중치는 10% 이내
- 키워드 사전과 하이브리드 적용 (단독 사용 금지)
- 추론 결과는 항상 **사람 검수 옵션** 제공

## Consequences

### 좋은 점

- **법적 안전성 확보**: 공공데이터는 공개·재활용이 허용되어 명예훼손/저작권 리스크 최소
- **신뢰성**: 사용자가 점수의 근거를 확인할 수 있어 점수에 대한 의심 감소
- **출시 속도**: Phase 2 MVP에서 공공데이터만으로 45점 점수 시스템 가동 가능 (AI/사용자 데이터 없이도)
- **차별화**: 광고 기반/크롤링 기반 경쟁 플랫폼과 명확히 구분되는 포지션
- **공익성**: [docs/principles.md](../principles.md) 의 신뢰 약속 7가지와 정합

### 나쁜 점 / 주의할 점

- **공공 API는 느리고 불안정**: 응답 시간 1~3초, 가끔 다운. **로컬 DB 캐싱 필수**
- **공공 API는 형식이 제각각**: XML/JSON 혼재, 필드명 비일관. **어댑터 레이어 필요**
- **공공 데이터의 갱신 주기가 느림**: 실거래가는 1개월 지연. 최신성 한계 명시 필요
- **공공 API는 사용량 한도가 있다**: 활용신청 시 일일 한도 확인 + 캐싱으로 호출 최소화
- **민간 사이트 데이터를 못 쓰면 정보가 부족할 수 있음**: 호갱노노 리뷰 수준의 풍부함은 단기간에 따라가기 어려움. 사용자 후기 시스템으로 점진 보완

### 영향 받는 컴포넌트

- `brokers/services/` — 외부 API 호출 어댑터 (신규 작성 예정)
- `brokers/models.py` — `PublicDataSnapshot` 같은 캐싱 모델 추가 예정
- `interactions/models.py:Review` — 가중치 제한 정책 적용
- `community/` — 익명 게시판 데이터의 점수 반영 가중치 제한
